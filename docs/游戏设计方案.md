# 简单战棋游戏 - 设计方案

## 项目概述
- **游戏引擎**：Godot 4.x
- **脚本语言**：GDScript
- **游戏类型**：回合制网格战棋
- **场景数量**：1个主场景（战斗场景）
- **单位**：1个玩家 + 2个敌人
- **地图**：10×8 网格棋盘，含障碍物
- **回合**：移动阶段 → 行动阶段，两段式

---

## 核心架构

### 场景结构
```
MainScene (管理全局)
├── FloorLayer (TileMapLayer — 地板)
├── ObstacleLayer (TileMapLayer — 障碍物)
├── HighlightLayer (TileMapLayer — 运行时动态高亮)
├── Character容器
│   ├── Player (Character节点，挂 PlayerController)
│   ├── Enemy1 (Character节点，挂 AIController)
│   └── Enemy2 (Character节点，挂 AIController)
├── GridRenderer (桥接层 — 坐标转换 + 点击射线检测)
├── UI容器
│   ├── PlayerInfoPanel (玩家信息：HP、防御状态)
│   ├── EnemyInfoPanel (敌人信息显示)
│   └── ActionMenu (行动菜单，分两阶段：移动确认 → 攻击/防御)
└── GameManager (游戏逻辑控制器)
```

> **三层架构**（行业标准做法）：
>
> | 层 | 组件 | 类型 | 职责 |
> |----|------|------|------|
> | 数据层 | `BattleGridData` | Resource（不继承Node） | 存 GridCell 数组、BFS 寻路、角色占位，**完全独立于渲染** |
> | 渲染层 | `TileMapLayer ×3` | Godot 内置 Node | FloorLayer=地板 / ObstacleLayer=障碍物 / HighlightLayer=运行时高亮 |
> | 桥接层 | `GridRenderer` | Node | 坐标转换 `world↔grid`、点击射线映射到格子、控制 HighlightLayer 写入高亮 |
>
> Player 和 Enemy 是同一个 `character.tscn` 实例化，区别只在于注入的 Controller。
> Character 不挂在 TileMap 下面——它们是独立 Sprite2D，由 GridRenderer 负责计算其世界坐标与格子坐标的对应。

### 架构分层示意
```
┌──────────────────────────────────────┐
│  BattleGridData (Resource)           │  ← 数据层：纯逻辑，不继承 Node
│  - cells: Array[Array[GridCell]]     │
│  - BFS get_move_range()              │
│  - 不碰 Godot API                     │
└────────────┬─────────────────────────┘
             │ 读写
┌────────────▼─────────────────────────────┐
│  GridRenderer (Node)                      │  ← 桥接层
│  - grid_to_world(pos) → Vector2          │
│  - world_to_grid(screen_click) → Vec2i   │
│  - highlight_cells(coords, color)         │  → 写入 HighlightLayer
│  - clear_highlights()                     │
└────────────┬────────────────────────────┘
             │ 写入
┌────────────▼─────────────────────────────┐
│  TileMapLayer ×3 (Godot Node)             │  ← 渲染层：硬件加速
│  - FloorLayer: 地板 tileset               │
│  - ObstacleLayer: 障碍物 tileset          │
│  - HighlightLayer: 高亮覆盖层（蓝/红 tile）│
└───────────────────────────────────────────┘
```

### 类系统设计

> **设计思路**：
> - **三层分离**：数据 (BattleGridData) → 桥接 (GridRenderer) → 渲染 (TileMap)，各改各的互不影响
> - Controller 与 Character 解耦：角色只负责属性与战斗，不关心谁在控制
> - 回合两段式：先移动（位置决策），再行动（攻击/防御决策）

#### 1. BattleGridData类（battle_grid_data.gd）
**类型**：`Resource`（不继承 Node，纯数据类）  
**职责**：地图数据的唯一真相源。网格数据、BFS 寻路、角色占位管理。

**数据结构**：
```gdscript
class_name GridCell
var pos: Vector2i              # 格子坐标
var is_walkable: bool          # 是否可行走（false = 障碍物）
var terrain_type: int          # 地形类型（0=平地，预留扩展）
var occupant: Character        # 站在该格上的角色（null = 空格）

class_name BattleGridData
extends Resource

var width: int  = 10
var height: int = 8
var cells: Array[Array]        # 实际是 cells[y][x]，存 GridCell
```

**核心方法**：
```gdscript
# === 格子查询 ===
- get_cell(pos: Vector2i) -> GridCell
- is_walkable(pos: Vector2i) -> bool           # 在界内、可行走、无人占据
- is_in_bounds(pos: Vector2i) -> bool

# === 邻居与寻路 ===
- get_neighbors(pos: Vector2i) -> Array[Vector2i]   # 四方向相邻坐标
- get_move_range(origin: Vector2i, range: int) -> Array[Vector2i]  # BFS
- get_attack_range(origin: Vector2i, range: int) -> Array[Vector2i] # 曼哈顿距离 ≤ range
- manhattan_distance(a: Vector2i, b: Vector2i) -> int

# === 角色占位 ===
- place_character(character: Character, pos: Vector2i)  # 放到格子上
- move_character(from: Vector2i, to: Vector2i)           # 移动：clear旧格 + 占新格
- remove_character(pos: Vector2i)                        # 角色死亡时清理
- get_characters_in_range(origin: Vector2i, range: int, team: Team) -> Array[Character]

# === 地图初始化 ===
- init_from_tilemap(obstacle_layer: TileMapLayer)  # 遍历 ObstacleLayer，有 tile 的位置 is_walkable=false
```

**BFS 移动范围**（纯数据层，不碰渲染）：
```gdscript
func get_move_range(origin: Vector2i, move_limit: int) -> Array[Vector2i]:
    var result: Array[Vector2i] = []
    var visited: Dictionary = {}         # Vector2i → bool
    var queue: Array = [{pos=origin, dist=0}]
    visited[origin] = true

    while queue.size() > 0:
        var cur = queue.pop_front()
        for neighbor in get_neighbors(cur.pos):
            if not visited.has(neighbor) and is_walkable(neighbor) and cur.dist < move_limit:
                visited[neighbor] = true
                queue.push_back({pos=neighbor, dist=cur.dist + 1})
                result.append(neighbor)
    return result
```

#### 1b. TileMapLayer 配置（Godot 4.3+）

Godot 4.3+ 中 `TileMap` 废弃，改为每个层一个 `TileMapLayer` 节点。

| 节点 | 用途 | 编辑方式 |
|------|------|---------|
| `FloorLayer` | 地板（灰色铺满 10×8） | Godot 编辑器手绘 |
| `ObstacleLayer` | 障碍物 | Godot 编辑器手绘，`BattleGridData.init_from_tilemap()` 读取 |
| `HighlightLayer` | 高亮覆盖层 | **运行时由 `GridRenderer` 写入** |

**所有 TileMapLayer 共用同一个 tileset**：

| 项目 | 值 |
|------|-----|
| tileset 文件 | 内嵌于 `main_scene.tscn` 的 `[sub_resource type="TileSet"]`，纹理使用 `assets/tilesets/map.png` |
| 高亮所在 source_id | `6` |
| 红色 atlas 坐标 | `(0, 0)` → `GridRenderer.ATLAS_ATTACK` |
| 蓝色 atlas 坐标 | `(1, 0)` → `GridRenderer.ATLAS_MOVE` |

> 障碍物层触发判定只看"这个格子有没有 tile"（`get_cell_source_id() != -1`），不关心具体哪个 source。

#### 1c. GridRenderer类（grid_renderer.gd）
**类型**：Node（挂在 MainScene）  
**职责**：桥接数据层和渲染层。坐标转换、点击检测、高亮 HighlightLayer。

```gdscript
class_name GridRenderer
extends Node

var grid_data: BattleGridData           # 注入的数据引用
var highlight_layer: TileMapLayer       # 注入的 HighlightLayer 引用
var highlight_source_id: int = 6        # tileset 中高亮图集的 source_id（当前项目常量）

# 同一 source 内的 atlas 坐标（二维）
const ATLAS_ATTACK = Vector2i(0, 0)     # 红色
const ATLAS_MOVE   = Vector2i(1, 0)     # 蓝色

# === 坐标转换（使用 HighlightLayer 作为参考坐标系，三层的坐标对齐） ===
func grid_to_world(grid_pos: Vector2i) -> Vector2:
    return highlight_layer.map_to_local(grid_pos)

func world_to_grid(world_pos: Vector2) -> Vector2i:
    return highlight_layer.local_to_map(world_pos)

# === 点击检测 ===
func get_clicked_cell(event: InputEventMouseButton) -> Vector2i:
    var local_pos = highlight_layer.to_local(event.position)
    return highlight_layer.local_to_map(local_pos)

# === 高亮控制 ===
func highlight_cells(coords: Array[Vector2i], atlas_pos: Vector2i):
    for cell in coords:
        highlight_layer.set_cell(cell, highlight_source_id, atlas_pos)

func clear_highlights():
    highlight_layer.clear()  # 清空 HighlightLayer
```

#### 3. Controller体系（controller.gd）

**基类 Controller**：
```gdscript
class_name Controller
extends Node

var battle_grid_data: BattleGridData   # GameManager 初始化时注入

func decide_move(character: Character) -> Vector2i:
    return character.grid_pos  # 默认原地不动

func decide_action(character: Character) -> ActionType:
    return ActionType.ATTACK

func decide_target(character: Character, enemies: Array[Character]) -> Character:
    return enemies[0]
```

**PlayerController（player_controller.gd）**：两段式 UI 输入
```gdscript
class_name PlayerController
extends Controller

var pending_move: Vector2i          # 移动阶段：玩家点击的格子
var pending_action: ActionType      # 行动阶段：玩家点的按钮
var pending_target: Character       # 行动阶段：玩家点的目标
var phase: int = PHASE_MOVE         # PHASE_MOVE=移动阶段, PHASE_WAIT=等待确认

    # 1. GridRenderer.highlight_cells(character.get_move_options(), ATLAS_MOVE)
    # 2. 等待玩家点击可走格子
    # 3. 返回目标坐标
    return pending_move

func decide_action(_character: Character) -> ActionType:
    return pending_action

func decide_target(_character: Character, enemies: Array[Character]) -> Character:
    return pending_target
```

**AIController（ai_controller.gd）**：包含移动和行动决策
```gdscript
class_name AIController
extends Controller

func decide_move(character: Character) -> Vector2i:
    var options = character.get_move_options()
    var enemies = get_alive_enemies()
    if enemies.is_empty(): return character.grid_pos
    var target = pick_priority_target(enemies)
    return get_closest_to(options, target.grid_pos)

func decide_action(character: Character) -> ActionType:
    if character.get_attack_targets().size() > 0:
        return ActionType.ATTACK
    return ActionType.ATTACK if randf() < 0.7 else ActionType.DEFEND

func decide_target(character: Character, enemies: Array[Character]) -> Character:
    return pick_priority_target(enemies)  # 优先血量最低的

func pick_priority_target(enemies: Array[Character]) -> Character:
    return enemies.reduce(func(a, b): return a if a.current_hp < b.current_hp else b)
```

#### 5. GameManager类（game_manager.gd）
**职责**：管理回合流程，串联移动阶段和行动阶段

**核心方法**：
```gdscript
- start_game()              # 创建BattleGridData → 从TileMap初始化 → 创建角色并站位 → 注入Controller
- start_round()             # 新回合：按顺序执行每个存活角色的回合
- execute_move_phase(actor: Character)   # 调用 controller.decide_move(actor)，通过 battle_grid_data 移动
- execute_action_phase(actor: Character)  # 调用 controller.decide_action(actor) + decide_target()，执行
- check_win_condition()     # 检查胜负
- end_game(winner)          # 游戏结束
```

**初始化顺序**：
```gdscript
func start_game():
    # 1. 创建数据层
    battle_grid_data = BattleGridData.new()
    battle_grid_data.init_from_tilemap($ObstacleLayer)
    # 2. 创建角色并站位
    player = create_character_at(Vector2i(2, 4), PlayerController.new(), PLAYER_STATS)
    enemy1 = create_character_at(Vector2i(5, 1), AIController.new(), ENEMY1_STATS)
    enemy2 = create_character_at(Vector2i(8, 1), AIController.new(), ENEMY2_STATS)
    # 3. 注入依赖
    for controller in [player.controller, enemy1.controller, enemy2.controller]:
        controller.battle_grid_data = battle_grid_data
    start_round()
```

**关键流程**（每回合两段式）：
```gdscript
func start_round():
    # === 玩家回合 ===
    execute_move_phase(player)      # controller.decide_move → battle_grid_data.move_character
    execute_action_phase(player)    # controller.decide_action + decide_target → combat_system.execute
    
    # === 敌人回合 ===
    for enemy in alive_enemies:
        execute_move_phase(enemy)
        execute_action_phase(enemy)
    
    check_win_condition()
```

## 回合制流程（两段式）

```
游戏初始化（BattleGridData 从 TileMap 读障碍物 + 角色站位）
  ↓
┌─ 玩家回合 ──────────────────────────────┐
│  [移动阶段]                              │
│  GridRenderer 高亮可走格子（蓝）          │
│  → 玩家点击目标格 → battle_grid_data 移动 │
│  [行动阶段]                              │
│  GridRenderer 高亮可攻击目标（红）        │
│  → 玩家选攻击/防御 → 选目标 → 执行       │
└──────────────────────────────────────────┘
  ↓
┌─ 敌人1 回合 ─────────────────────────┐
│  [移动阶段] AI 计算 → 移动到最近目标  │
│  [行动阶段] AI 判断 → 攻击/防御      │
└──────────────────────────────────────┘
  ↓
┌─ 敌人2 回合 ───────（同上）──────────┐
  ↓
检查胜负条件
  ├─ 玩家血量 ≤ 0 → 游戏失败 → 结束
  ├─ 所有敌人血量 ≤ 0 → 游戏胜利 → 结束
  └─ 都活着 → 下一回合
```

> **移动阶段**：必须先移动（可以留在原地），移动结束后不可反悔。
> **行动阶段**：到达位置后，攻击范围由新位置决定。

## 技能系统

### 移动
- **消耗**：每回合1次（先于行动）
- **移动范围**：由 `move_range` 决定（默认 3 格）
- **限制**：不能穿过障碍物或占据的格子
- **高亮**：蓝色 = 可移动到的格子，白色闪烁 = 当前选中目标

### 技能设定

#### 攻击
- **消耗**：每回合1次（移动后）
- **范围**：近战相邻格（曼哈顿距离 ≤ 1），可后续扩展远程
- **效果**：对目标造成伤害
- **伤害计算**：`基础伤害 = 攻击力 - 敌方防御力`
- **防御减伤**：若目标防御中，伤害减少 50%
- **最低伤害**：1 点（确保能逐渐消耗敌人）

#### 防御
- **消耗**：每回合1次（移动后）
- **效果**：进入防御状态
- **防御加成**：本回合及下一回合受到的伤害减少 50%
- **持续时间**：直到该角色下一次行动开始时重置

#### 等待
- **效果**：不执行任何操作（占位，可后续扩展为"蓄力"等）

## UI设计

### 网格地图
- **格子尺寸**：64×64 px
- **地图大小**：10×8（宽×高），可后续扩展
- **格子外观**：浅色=可行走，深色=障碍物
- **高亮颜色**：
  - 🔵 蓝色 = 可移动范围
  - 🔴 红色 = 可攻击范围
  - ⚪ 白色闪烁 = 当前选中目标/目的地
- **交互**：点击高亮格子 = 确认移动目标 / 确认攻击目标

### 玩家操作面板（两段式）

**移动阶段**：
- 提示文字："请选择移动目标"
- 按钮："原地不动"（跳过移动）
- 点击蓝色格子 → 角色移动到该位置 → 进入行动阶段

**行动阶段**：
- **HP显示**：`玩家名 HP: 80/100`
- **防御状态指示**：当防御时显示盾牌图标或文字"防御中"
- **行动按钮**：

  - 按钮3："⏳ 等待" → 跳过行动

- **禁用逻辑**：行动执行后，所有按钮禁用直到下回合

## 实现步骤（推荐顺序）

### 阶段1：数据层 + 渲染层（最先做）
1. 创建 `GridCell` 和 `BattleGridData`（Resource，纯数据）
2. 实现 BFS `get_move_range()`
3. 在 Godot 编辑器中创建 3 个 TileMapLayer + tilesets（地板、障碍物、高亮）
4. 手绘 FloorLayer（铺满）和 ObstacleLayer（3块障碍物）
5. 实现 `BattleGridData.init_from_tilemap()`：读取 ObstacleLayer 生成障碍物数据

### 阶段2：桥接层
1. 实现 `GridRenderer`（grid_to_world / world_to_grid）
2. 点击检测：鼠标点击 → 格子坐标
3. 高亮控制：`highlight_cells()` 写 HighlightLayer

### 阶段3：角色基础
1. 创建 Character 类（加 grid_pos、move_range、attack_range）
2. `get_move_options()` 委托 `battle_grid_data.get_move_range()`
3. 角色 Sprite2D 渲染，坐标跟随 `grid_pos`

### 阶段4：移动系统
1. 点击角色 → GridRenderer 高亮可走范围（蓝）
2. 点击蓝色格子 → 移动角色 → 清除高亮
3. 障碍物和占位检测通过 `BattleGridData.is_walkable()`

### 阶段5：Controller 体系
1. Controller 基类（decide_move + decide_action + decide_target）
2. PlayerController（两段式：移动读点击 → 行动读按钮）
3. AIController（向最近目标移动 + 攻击决策）

### 阶段6：战斗系统
1. CombatSystem（曼哈顿距离范围判定 + 伤害计算）
2. 攻击范围高亮（红）
3. 攻击目标选择交互

### 阶段7：GameManager 回合流程
1. `start_game()`：创建 BattleGridData → 从 TileMap 初始化 → 角色站位 → 注入 Controller
2. 两段式回合：`execute_move_phase` → `execute_action_phase`
3. 胜负判定

### 阶段8：UI 和打磨
1. ActionMenu（两段式按钮）
2. InfoPanel（HP、状态显示）
3. 战斗日志
4. 数值测试和平衡调整

## 技术亮点
- ✅ 三层架构（工业标准）：数据层 (Resource) → 桥接层 (GridRenderer) → 渲染层 (TileMap)，各层独立修改
- ✅ 数据驱动：TileMap 只是渲染器，所有逻辑在 BattleGridData 上运算，为性能可扩展打底
- ✅ 组件化设计：Controller 与 Character 解耦
- ✅ 依赖注入：角色创建时动态赋予 Controller 和数据层引用
- ✅ 策略模式：AI/玩家输入互换，不影响角色本体
- ✅ BFS 寻路：纯数据层运算，不碰场景树
- ✅ 两段式回合：移动→行动分离，后续可扩展连击、技能链
- ✅ 编辑器友好：障碍物在 Godot 编辑器里手绘，不用写代码生成地图
- ✅ 易于扩展（新地形、多层渲染效果、新AI、新技能、大尺寸地图）
