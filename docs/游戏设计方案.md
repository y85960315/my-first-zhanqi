# 简单战棋游戏 - 设计方案

## 项目概述
- **游戏引擎**：Godot 4.x
- **脚本语言**：GDScript
- **游戏类型**：回合制网格战棋
- **场景数量**：1个主场景（战斗场景）
- **单位**：1个玩家 + 2个敌人
- **地图**：网格棋盘，每个单位占据一个格子
- **回合**：移动阶段 → 行动阶段，两段式

---

## 核心架构

### 场景结构
```
MainScene (管理全局)
├── GridMap (网格地图 — 渲染格子、高亮可走/攻击范围)
│   ├── Player (Character节点，挂 PlayerController)
│   ├── Enemy1 (Character节点，挂 AIController)
│   └── Enemy2 (Character节点，挂 AIController)
├── UI容器
│   ├── PlayerInfoPanel (玩家信息：HP、防御状态)
│   ├── EnemyInfoPanel (敌人信息显示)
│   └── ActionMenu (行动菜单，分两阶段：移动确认 → 攻击/防御)
└── GameManager (游戏逻辑控制器)
```

> Player 和 Enemy 本质是同一个 Character 场景，区别只在于注入的 Controller。场景中只需一个 `character.tscn`，GameManager 初始化时动态设置属性、坐标并挂对应的 Controller。
>
> GridMap 是独立节点，负责地图数据的存储与渲染。Character 作为 GridMap 的子节点，坐标与格子对齐。

### 类系统设计

> **设计思路**：
> - Controller 与 Character 解耦：角色只负责属性与战斗，不关心谁在控制
> - GridMap 独立于角色：地图是纯数据结构，角色只是"放在"某个格子上
> - 回合两段式：先移动（位置决策），再行动（攻击/防御决策）

#### 1. GridMap类（grid_map.gd）
**职责**：网格地图的数据存储、BFS寻路、高亮渲染

**数据结构**：
```gdscript
class_name GridCell
var pos: Vector2i           # 格子坐标
var is_walkable: bool       # 是否可行走
var terrain_type: int       # 地形类型（0=平地，可后续扩展）
var occupant: Character     # 站在该格上的角色（null=空格）
```

**核心方法**：
```gdscript
- get_cell(pos: Vector2i) -> GridCell
- is_walkable(pos: Vector2i) -> bool          # 可行走且无人占据
- get_neighbors(pos: Vector2i) -> Array[Vector2i]  # 四方向相邻坐标
- get_move_range(origin: Vector2i, range: int) -> Array[Vector2i]  # BFS 计算可移动范围
- get_attack_range(origin: Vector2i, range: int) -> Array[Vector2i] # 计算攻击范围
- move_character(from: Vector2i, to: Vector2i)  # 移动角色到新格子
- highlight_cells(cells: Array[Vector2i], color: Color)  # 高亮渲染
- clear_highlights()
```

**BFS 移动范围**：
```gdscript
func get_move_range(origin: Vector2i, range: int) -> Array[Vector2i]:
    var result: Array[Vector2i] = []
    var visited = {}  # Dictionary[Vector2i, bool]
    var queue = [{pos=origin, dist=0}]
    visited[origin] = true
    while queue.size() > 0:
        var cur = queue.pop_front()
        for neighbor in get_neighbors(cur.pos):
            if not visited.has(neighbor) and is_walkable(neighbor) and cur.dist < range:
                visited[neighbor] = true
                queue.push_back({pos=neighbor, dist=cur.dist+1})
                result.append(neighbor)
    return result
```

**GridMap 数据示例**（5×5 网格）：
```
  ■ ■ ■ ■ ■        ■ = 可行走
  ■ ■ ■ ■ ■        □ = 不可行走（障碍）
  ■ ■ □ ■ ■
  ■ ■ ■ ■ ■
  ■ ■ ■ ■ ■
```

**核心属性**：

**核心方法**：

**基类 Controller**：

**AIController（ai_controller.gd）**：包含移动和行动决策
```gdscript
class_name AIController
extends Controller

func decide_move(character: Character) -> Vector2i:
    var options = character.get_move_options(grid_map)
    var enemies = get_alive_enemies()
    if enemies.is_empty(): return character.grid_pos
    var target = pick_priority_target(enemies)
    # 向最近目标移动：选 options 中离 target 最近的格子
    return get_closest_to(options, target.grid_pos)

func decide_action(character: Character) -> ActionType:
    # 如果攻击范围内有敌人 → 攻击，否则70%/30%
    if character.get_attack_options(grid_map).size() > 0:
        return ActionType.ATTACK
    return ActionType.ATTACK if randf() < 0.7 else ActionType.DEFEND

func decide_target(character: Character, enemies: Array[Character]) -> Character:
    return pick_priority_target(enemies)  # 优先血量最低的

func pick_priority_target(enemies: Array[Character]) -> Character:
    return enemies.reduce(func(a, b): return a if a.current_hp < b.current_hp else b)
```

#### 5. GameManager类（game_manager.gd）
**职责**：管理回合流程，串联移动阶段和行动阶段

**核心方法**：
```gdscript
- start_game()              # 初始化地图、角色站位、创建Controller并注入grid_map引用
- start_round()             # 新回合：按顺序执行每个存活角色的回合
- execute_move_phase(actor: Character)   # 调用 controller.decide_move(actor)，移动角色
- execute_action_phase(actor: Character)  # 调用 controller.decide_action(actor) + decide_target()，执行
- check_win_condition()     # 检查胜负
- end_game(winner)          # 游戏结束
```

**关键流程**（每回合两段式）：
```gdscript
func start_round():
    # === 玩家回合 ===
    execute_move_phase(player)          # 调用 player.controller.decide_move(player) → 移动
    execute_action_phase(player)         # 调用 player.controller.decide_action(player) + decide_target() → 执行
    
    # === 敌人回合 ===
    for enemy in alive_enemies:
        execute_move_phase(enemy)        # enemy.controller.decide_move(enemy) → 移动
        execute_action_phase(enemy)      # enemy.controller.decide_action(enemy) + decide_target() → 执行
    
    check_win_condition()
```

## 回合制流程（两段式）

```
游戏初始化（生成地图 + 角色站位）
  ↓
┌─ 玩家回合 ──────────────────────────┐
│  [移动阶段]                          │
│  高亮可走格子 → 玩家点击目标格 → 移动 │
│  [行动阶段]                          │
│  高亮可攻击目标 → 玩家选攻击/防御     │
│  → 选目标（若攻击）→ 执行行动        │
└──────────────────────────────────────┘
  ↓
┌─ 敌人1 回合 ─────────────────────────┐
│  [移动阶段] AI 计算 → 移动到最近目标  │
│  [行动阶段] AI 判断 → 攻击/防御      │
└──────────────────────────────────────┘
  ↓
┌─ 敌人2 回合 ───────（同上）──────────┐
  ↓
检查胜负条件
  ├─ 玩家血量 ≤ 0 → 游戏失败 → 结束
  ├─ 所有敌人血量 ≤ 0 → 游戏胜利 → 结束
  └─ 都活着 → 下一回合
```

> **移动阶段**：必须先移动（可以留在原地），移动结束后不可反悔。
> **行动阶段**：到达位置后，攻击范围由新位置决定。

## 技能系统

### 移动
- **消耗**：每回合1次（先于行动）
- **移动范围**：由 `move_range` 决定（默认 3 格）
- **限制**：不能穿过障碍物或占据的格子
- **高亮**：蓝色 = 可移动到的格子，白色闪烁 = 当前选中目标

### 技能设定

#### 攻击
- **消耗**：每回合1次（移动后）
- **范围**：近战相邻格（曼哈顿距离 ≤ 1），可后续扩展远程
- **效果**：对目标造成伤害
- **伤害计算**：`基础伤害 = 攻击力 - 敌方防御力`
- **防御减伤**：若目标防御中，伤害减少 50%
- **最低伤害**：1 点（确保能逐渐消耗敌人）

#### 防御
- **消耗**：每回合1次（移动后）
- **效果**：进入防御状态
- **防御加成**：本回合及下一回合受到的伤害减少 50%
- **持续时间**：直到该角色下一次行动开始时重置

#### 等待
- **效果**：不执行任何操作（占位，可后续扩展为"蓄力"等）

## UI设计

### 网格地图
- **格子尺寸**：64×64 px
- **地图大小**：初版 5×5（可后续扩展）
- **格子外观**：浅色=可行走，深色=障碍物
- **高亮颜色**：
  - 🔵 蓝色 = 可移动范围
  - 🔴 红色 = 可攻击范围
  - ⚪ 白色闪烁 = 当前选中目标/目的地
- **交互**：点击高亮格子 = 确认移动目标 / 确认攻击目标

### 玩家操作面板（两段式）

**移动阶段**：
- 提示文字："请选择移动目标"
- 按钮："原地不动"（跳过移动）
- 点击蓝色格子 → 角色移动到该位置 → 进入行动阶段

**行动阶段**：
- **HP显示**：`玩家名 HP: 80/100`
- **防御状态指示**：当防御时显示盾牌图标或文字"防御中"
- **行动按钮**：
  - 按钮1："⚔️ 攻击" → 进入目标选择模式（高亮可攻击敌人）
  - 按钮2："🛡️ 防御" → 立即执行
  - 按钮3："⏳ 等待" → 跳过行动
- **目标选择**：点击红色高亮的敌人 → 执行攻击
- **禁用逻辑**：行动执行后，所有按钮禁用直到下回合

### 敌人信息面板
- **Enemy1 HP**: `敌1 HP: 60/80`
- **Enemy2 HP**: `敌2 HP: 70/80`
- **战斗日志**：显示上回合的行动结果
  - 示例：`Enemy1 移动到(3,2)，攻击了你，造成12点伤害！`
  - 示例：`你移动到(2,1)，攻击了Enemy2，造成8点伤害！`

## 技术亮点
- ✅ 组件化设计：Controller 与 Character 解耦，GridMap 独立于角色
- ✅ 依赖注入：角色创建时动态赋予控制逻辑和初始坐标
- ✅ 策略模式：AI/玩家输入互换，不影响角色本体
- ✅ BFS 寻路：网格移动范围计算
- ✅ 两段式回合：移动→行动分离，后续可扩展连击、技能链
- ✅ 模块化代码结构
- ✅ 易于扩展（新地形、新AI、新技能、新角色类型）
